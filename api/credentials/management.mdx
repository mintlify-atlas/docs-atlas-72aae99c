## List Credentials

Retrieve credentials filtered by type, department, or unit.

### Endpoint

```http
GET /credenciales
```

### Query Parameters

<ParamField query="idTipo" type="integer" optional>
  Filter by credential type ID
</ParamField>

<ParamField query="idDepartamento" type="integer" optional>
  Filter by department ID
</ParamField>

<ParamField query="idUnidad" type="integer" optional>
  Filter by unit ID
</ParamField>

### Response

<ResponseField name="data" type="array">
  <Expandable title="CuentaEntityDto">
    <ResponseField name="id" type="integer">
      Unique credential identifier
    </ResponseField>
    
    <ResponseField name="tipoId" type="integer">
      Credential type ID
    </ResponseField>
    
    <ResponseField name="tipoNombre" type="string">
      Credential type name
    </ResponseField>
    
    <ResponseField name="departamentoId" type="integer">
      Department ID
    </ResponseField>
    
    <ResponseField name="departamentoNombre" type="string">
      Department name
    </ResponseField>
    
    <ResponseField name="unidadId" type="integer">
      Unit ID
    </ResponseField>
    
    <ResponseField name="unidadClave" type="string">
      Unit code/identifier
    </ResponseField>
    
    <ResponseField name="unidadNombreCompleto" type="string">
      Full unit name
    </ResponseField>
    
    <ResponseField name="usuario" type="string">
      Username/account identifier
    </ResponseField>
    
    <ResponseField name="clave" type="string">
      Password (may be encrypted or masked)
    </ResponseField>
    
    <ResponseField name="nota" type="string">
      Additional notes about this credential
    </ResponseField>
    
    <ResponseField name="creado" type="string">
      Creation timestamp (ISO 8601)
    </ResponseField>
    
    <ResponseField name="actualizado" type="string">
      Last update timestamp (ISO 8601)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="message" type="string">
  Success message: "Cuentas registradas"
</ResponseField>

### Example Request

```bash cURL
curl -X GET "http://localhost:8081/comialex/api/integra/credenciales?idDepartamento=5&idUnidad=3" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

### Example Response

```json
{
  "data": [
    {
      "id": 42,
      "tipoId": 1,
      "tipoNombre": "Email Corporativo",
      "departamentoId": 5,
      "departamentoNombre": "Tecnología",
      "unidadId": 3,
      "unidadClave": "TI-001",
      "unidadNombreCompleto": "Tecnología - Desarrollo",
      "usuario": "jperez@company.com",
      "clave": "********",
      "nota": "Cuenta principal de desarrollo",
      "creado": "2025-01-15T10:30:00",
      "actualizado": "2025-02-20T14:45:00"
    }
  ],
  "message": "Cuentas registradas",
  "success": true
}
```

## Create Credential

Register a new credential account.

### Endpoint

```http
POST /credenciales
```

### Request Body

<ParamField body="usuario" type="string" required>
  Username or account identifier. Cannot be empty.
</ParamField>

<ParamField body="clave" type="string" optional>
  Password or access key for the credential
</ParamField>

<ParamField body="idTipoCuenta" type="integer" required>
  ID of the credential type
</ParamField>

<ParamField body="idDepartamento" type="integer" required>
  ID of the department this credential is associated with
</ParamField>

<ParamField body="idUnidad" type="integer" required>
  ID of the unit this credential belongs to
</ParamField>

<ParamField body="nota" type="string" optional>
  Additional notes or description
</ParamField>

### Response

<ResponseField name="data" type="null">
  No data returned on success
</ResponseField>

<ResponseField name="message" type="string">
  Success message: "Cuenta registrada"
</ResponseField>

### Example Request

```bash cURL
curl -X POST http://localhost:8081/comialex/api/integra/credenciales \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "usuario": "jperez@company.com",
    "clave": "SecurePass123!",
    "idTipoCuenta": 1,
    "idDepartamento": 5,
    "idUnidad": 3,
    "nota": "Cuenta principal de desarrollo"
  }'
```

### Example Response

```json
{
  "data": null,
  "message": "Cuenta registrada",
  "success": true
}
```

## Update Credential

Update an existing credential.

### Endpoint

```http
PUT /credenciales/{id}
```

### Path Parameters

<ParamField path="id" type="integer" required>
  Credential ID to update
</ParamField>

### Request Body

Same structure as Create Credential endpoint.

<ParamField body="usuario" type="string" required>
  Username or account identifier
</ParamField>

<ParamField body="clave" type="string" optional>
  Password or access key
</ParamField>

<ParamField body="idTipoCuenta" type="integer" required>
  Credential type ID
</ParamField>

<ParamField body="idDepartamento" type="integer" required>
  Department ID
</ParamField>

<ParamField body="idUnidad" type="integer" required>
  Unit ID
</ParamField>

<ParamField body="nota" type="string" optional>
  Additional notes
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message: "Cuenta actualizada"
</ResponseField>

### Example Request

```bash cURL
curl -X PUT http://localhost:8081/comialex/api/integra/credenciales/42 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "usuario": "jperez@company.com",
    "clave": "NewSecurePass456!",
    "idTipoCuenta": 1,
    "idDepartamento": 5,
    "idUnidad": 3,
    "nota": "Password actualizado"
  }'
```

## Delete Credential Type

Delete a credential type.

### Endpoint

```http
DELETE /credenciales/tipo/{id}
```

### Path Parameters

<ParamField path="id" type="integer" required>
  Credential type ID to delete
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success message: "Tipo eliminado"
</ResponseField>

### Example Request

```bash cURL
curl -X DELETE http://localhost:8081/comialex/api/integra/credenciales/tipo/4 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

<Warning>
Deleting a credential type may affect existing credentials that reference this type. Ensure no credentials are using this type before deletion.
</Warning>
